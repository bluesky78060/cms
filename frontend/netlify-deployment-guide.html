🚀 Netlify 무료 호스팅

완벽 배포 가이드

건축 견적/청구 시스템을 3분만에 온라인 배포하기


차근차근 따라하면 누구나 성공! 👍

📋 배포 과정 개요

  • GitHub 계정 및 저장소 준비
  • Netlify 회원가입 및 로그인
  • GitHub와 Netlify 연결
  • 배포 설정 구성
  • 자동 배포 및 확인
소요시간: 약 3-5분 ⏰
비용: 완전 무료 💰
난이도: 초보자도 OK! 😊

📝 준비사항 체크

  • GitHub 계정 (github.com)
  • 프로젝트가 GitHub에 업로드됨
  • package.json에 build 스크립트 존재
  • 인터넷 연결 상태 양호
✅ 우리 프로젝트는 모든 준비가 완료되었습니다!

🌐 1단계: Netlify 접속

1

웹 브라우저에서 다음 주소로 이동하세요

https://www.netlify.com
Netlify 홈페이지 스크린샷
(파란색 "Sign up" 버튼이 보임)

오른쪽 상단의 "Sign up" 버튼을 클릭하세요

📝 2단계: GitHub로 회원가입

2

"GitHub" 버튼을 클릭하여 회원가입하세요

GitHub 로그인 옵션 스크린샷
(GitHub 아이콘과 "Continue with GitHub" 버튼)
💡 GitHub 계정으로 가입하면 저장소 연결이 자동으로 됩니다!

🔐 3단계: GitHub 권한 승인

3

GitHub에서 Netlify 접근 권한을 요청합니다

  • "Authorize netlify" 버튼 클릭
  • 필요시 GitHub 비밀번호 재입력
⚠️ 이 단계는 안전합니다. Netlify가 코드를 읽어서 배포하기 위해 필요한 권한입니다.

🏗️ 4단계: 새 사이트 생성

4

Netlify 대시보드에서 다음을 클릭하세요

  • "Add new site" 버튼 클릭
  • "Import an existing project" 선택
Netlify 대시보드 스크린샷
(초록색 "Add new site" 버튼)

📁 5단계: GitHub 저장소 선택

5

"GitHub" 옵션을 선택하세요

Git 공급자 선택 화면
(GitHub, GitLab, Bitbucket 옵션들)

저장소 목록에서 "construction-management-system"을 찾아 클릭

⚙️ 6단계: 빌드 설정 구성

6

저장소를 선택하면 설정 화면이 나타납니다

"Site settings and deploy" 화면
(Branch to deploy, Build settings 섹션이 보임)

다음 3개 입력란을 정확히 채우세요:

  • Base directory: frontend
  • Build command: npm run build
  • Publish directory: frontend/build
⚠️ 각 입력란을 클릭하고 정확히 타이핑하세요!

📝 6단계 상세: 각 입력란 설명

6

1️⃣ Base directory (기본 디렉토리)

frontend

→ 프로젝트의 시작 폴더를 지정합니다

2️⃣ Build command (빌드 명령어)

npm run build

→ 프로젝트를 빌드할 때 실행할 명령어

3️⃣ Publish directory (배포 폴더)

frontend/build

→ 빌드된 파일들이 저장되는 폴더

💡 이 설정들은 우리 프로젝트 구조에 맞게 최적화된 값입니다!

🔧 7단계: 환경 변수 설정

7

"Show advanced" 클릭 후 환경 변수 추가:

NODE_VERSION = 18 NPM_FLAGS = --legacy-peer-deps
💡 이 설정은 빌드 오류를 방지하고 최신 Node.js 버전을 사용합니다

🚀 8단계: 배포 시작

8

"Deploy site" 버튼을 클릭하세요

Deploy site 버튼 스크린샷
(큰 파란색 버튼)

Netlify가 자동으로 다음 작업을 수행합니다:

  • 코드 다운로드
  • 의존성 설치 (npm install)
  • 프로젝트 빌드 (npm run build)
  • 배포 완료

⏳ 9단계: 배포 진행 상황 확인

9

배포 과정을 실시간으로 확인할 수 있습니다

  • 노란색: 배포 진행 중 🟡
  • 초록색: 배포 성공 🟢
  • 빨간색: 배포 실패 🔴
⏰ 일반적으로 2-3분 소요됩니다
처음 배포시에는 조금 더 걸릴 수 있어요

🎉 10단계: 배포 완료 및 확인

10

배포가 완료되면 다음을 확인하세요:

  • 자동 생성된 URL (예: https://wonderful-site-123456.netlify.app)
  • 초록색 "Published" 상태
  • 사이트 미리보기 링크
🎊 축하합니다! 성공적으로 배포되었습니다!

🏷️ 11단계: 사이트 이름 변경 (선택사항)

11

기본 생성된 이름 대신 원하는 이름으로 변경하세요

  • "Site settings" 클릭
  • "Change site name" 선택
  • 원하는 이름 입력 (예: my-construction-system)
최종 URL: https://my-construction-system.netlify.app

🔄 12단계: 자동 배포 확인

12

이제부터 GitHub에 코드를 푸시하면 자동으로 배포됩니다!

  • 코드 수정 후 GitHub에 커밋
  • Netlify가 자동으로 감지
  • 새 버전 자동 배포
  • 실시간 업데이트 완료
🎯 완전 자동화된 배포 시스템 구축 완료!

🎊 배포 완료!

수고하셨습니다!
