🏗️ 건설업 관리 시스템

윈도우 컴퓨터 배포 가이드

완전한 단계별 설치 및 배포 매뉴얼

Node.js → 프로젝트 설정 → 빌드 → 배포 → 문제 해결

📋 배포 가이드 개요

목표: React 기반 건설업 관리 시스템을 윈도우 환경에서 실행 가능하도록 배포

🎯 주요 배포 방법 3가지

  1. 개발 서버 방식 - 개발/테스트 환경 (npm start)
  2. 정적 웹사이트 방식 - 웹서버 배포 (npm build)
  3. Electron 데스크톱 앱 - 독립 실행 프로그램
✅ 이 가이드를 따라하면 기술 지식이 없는 사용자도 쉽게 설치할 수 있습니다!

🛠️ 사전 요구사항

시스템 요구사항

  • 운영체제: Windows 10/11 (64bit 권장)
  • 메모리: 최소 4GB RAM (8GB 권장)
  • 저장공간: 최소 2GB 여유공간
  • 네트워크: 인터넷 연결 (초기 설치 시)

필수 소프트웨어

  1. Node.js 18.x 이상 - JavaScript 런타임
  2. Git for Windows - 소스코드 다운로드용
  3. Visual Studio Code - 텍스트 에디터 (선택사항)
⚠️ 중요: 모든 소프트웨어는 "관리자 권한"으로 설치해야 합니다!

📥 1단계: Node.js 설치

1
Node.js 공식 사이트 접속
https://nodejs.org/ 에서 LTS 버전 다운로드
2
설치 파일 실행
node-v18.x.x-x64.msi 파일을 관리자 권한으로 실행
3
설치 확인
명령 프롬프트(cmd)에서 다음 명령어 실행:
node --version npm --version
✅ 버전 정보가 나타나면 설치 완료! (예: v18.17.0, 9.6.7)

🔧 2단계: Git 설치

1
Git for Windows 다운로드
https://git-scm.com/download/win 에서 다운로드
2
설치 옵션 설정
• "Git from the command line and also from 3rd-party software" 선택
• "Use Windows' default console window" 선택
• 나머지는 기본값 사용
3
설치 확인
새로운 명령 프롬프트에서:
git --version
💡 팁: Git Bash도 함께 설치되므로 Unix 스타일 명령어 사용 가능

⬇️ 3단계: 프로젝트 다운로드

방법 1: Git Clone (권장)

1
작업 폴더 생성
예: C:\Projects\ 폴더 생성
2
명령 프롬프트에서 폴더 이동
cd C:\Projects git clone [저장소 URL] construction-management-system

방법 2: ZIP 파일 다운로드

  1. GitHub/GitLab에서 "Download ZIP" 클릭
  2. 압축 해제하여 C:\Projects\construction-management-system 에 저장
프로젝트 구조: C:\Projects\construction-management-system\ ├── frontend/ # React 웹 애플리케이션 ├── backend/ # FastAPI 서버 (선택사항) ├── README.md # 프로젝트 설명 └── package.json # 의존성 정보

📦 4단계: 의존성 설치

1
프론트엔드 폴더로 이동
cd C:\Projects\construction-management-system\frontend
2
패키지 설치 (시간 소요: 3-5분)
npm install
⚠️ 주의사항:
• 인터넷 연결 필수
• 바이러스 백신이 설치를 방해할 수 있음
• node_modules 폴더가 생성됨 (용량: ~200MB)
3
설치 완료 확인
package-lock.json 파일과 node_modules 폴더 생성 확인

🚀 5단계: 개발 서버 실행

개발 서버 방식: 개발/테스트 환경, 실시간 코드 변경 반영
1
개발 서버 시작
npm start
2
자동 브라우저 열림
http://localhost:3000 에서 애플리케이션 실행
3
서버 중지
명령 프롬프트에서 Ctrl + C
✅ 성공! 웹 브라우저에서 건설업 관리 시스템이 열리면 정상 작동

🏗️ 6단계: 프로덕션 빌드

프로덕션 빌드: 최적화된 정적 파일 생성, 웹서버 배포용
1
빌드 실행 (시간 소요: 1-2분)
npm run build
2
빌드 결과 확인
frontend/build/ 폴더 생성 및 파일들 확인
빌드 결과물: frontend/build/ ├── index.html # 메인 HTML 파일 ├── static/ │ ├── css/ # 최적화된 CSS 파일들 │ └── js/ # 최적화된 JavaScript 파일들 ├── favicon.ico # 파비콘 └── manifest.json # 웹앱 매니페스트
✅ build 폴더의 모든 파일을 웹서버에 업로드하면 배포 완료!

🌐 7단계: 로컬 웹서버 실행

방법 1: serve 패키지 사용 (권장)

1
serve 패키지 전역 설치
npm install -g serve
2
빌드 폴더에서 서버 시작
serve -s build -p 3000

방법 2: Python HTTP 서버

cd build python -m http.server 8000
💡 포트 설명:
• 3000: 일반적인 React 개발 서버 포트
• 8000: Python HTTP 서버 기본 포트
• 80: 웹서버 표준 포트 (관리자 권한 필요)

🖥️ 8단계: Electron 데스크톱 앱 생성

Electron: 웹 기술로 네이티브 데스크톱 앱 생성
1
Electron 관련 패키지 설치
npm install --save-dev electron electron-builder concurrently wait-on
2
개발용 Electron 실행
npm run electron-dev
3
Windows용 실행 파일 생성
npm run dist-win
✅ dist/ 폴더에 .exe 설치 파일 생성! 더블클릭으로 설치 가능

🌍 9단계: 네트워크 배포 (사무실 내 공유)

로컬 네트워크에서 접근 가능하게 설정

1
본인 컴퓨터의 IP 주소 확인
ipconfig
2
개발 서버를 네트워크에 오픈
npm start -- --host 0.0.0.0
3
다른 컴퓨터에서 접근
http://[서버IP]:3000 (예: http://192.168.1.100:3000)
⚠️ 보안 주의사항:
• 방화벽에서 3000 포트 허용 필요
• 신뢰할 수 있는 네트워크에서만 사용
• 외부 인터넷 접근 차단 권장

⚙️ 10단계: Windows 서비스로 등록

Windows 서비스: 컴퓨터 부팅 시 자동으로 시작되는 백그라운드 프로세스
1
node-windows 패키지 설치
npm install -g node-windows
2
서비스 스크립트 작성
service-install.js 파일 생성:
var Service = require('node-windows').Service; var svc = new Service({ name: '건설업 관리 시스템', description: 'Construction Management System Web Server', script: 'C:\\Projects\\construction-management-system\\frontend\\server.js' }); svc.on('install', function(){ svc.start(); }); svc.install();
3
관리자 권한으로 실행
node service-install.js

🗄️ 11단계: 데이터베이스 설정 (선택사항)

현재는 localStorage 사용 중이지만, 향후 확장을 위한 데이터베이스 준비

SQLite (간단한 파일 기반 DB)

1
SQLite 패키지 설치
npm install sqlite3

PostgreSQL (본격적인 DB)

1
PostgreSQL 다운로드 및 설치
https://www.postgresql.org/download/windows/
2
데이터베이스 생성
createdb -U postgres construction_db
⚠️ 현재 시스템은 localStorage를 사용하므로 데이터베이스 없이도 완전히 동작합니다!

🔒 12단계: 보안 설정

방화벽 설정

1
Windows 방화벽 설정
제어판 → 시스템 및 보안 → Windows Defender 방화벽
2
인바운드 규칙 추가
포트 3000번에 대한 규칙 추가 (로컬 네트워크만)

HTTPS 설정 (선택사항)

npm install --save-dev https-localhost
🚨 보안 체크리스트:
✅ 불필요한 포트 차단
✅ 정기적인 보안 업데이트
✅ 강력한 관리자 비밀번호 설정
✅ 외부 접근 제한
💡 팁: 현재 시스템은 인증 기능이 없으므로 사내 네트워크에서만 사용 권장

🤖 13단계: 자동화 스크립트 작성

배치 파일로 쉬운 실행

1
시작 스크립트 작성
start-server.bat 파일 생성:
@echo off echo 건설업 관리 시스템 시작 중... cd /d "C:\Projects\construction-management-system\frontend" call npm start pause
2
중지 스크립트 작성
stop-server.bat 파일 생성:
@echo off echo 서버 중지 중... taskkill /f /im node.exe echo 서버가 중지되었습니다. pause
3
바탕화면 바로가기 생성
.bat 파일들을 바탕화면에 바로가기로 생성
✅ 이제 더블클릭 한 번으로 시스템 시작/중지 가능!

🔧 14단계: 문제 해결

❌ 문제: "node를 인식할 수 없습니다"

해결방법:
1. 명령 프롬프트 재시작
2. 시스템 환경 변수에서 Node.js 경로 확인
3. Node.js 재설치

❌ 문제: "npm install 실패"

해결방법:
1. 바이러스 백신 일시 비활성화
2. npm 캐시 삭제: npm cache clean --force
3. node_modules 삭제 후 재설치

❌ 문제: "포트 3000 이미 사용 중"

해결방법:
1. 다른 포트 사용: npm start -- --port 3001
2. 프로세스 종료: taskkill /f /im node.exe

❌ 문제: "권한 부족 오류"

해결방법:
1. 명령 프롬프트를 관리자 권한으로 실행
2. 프로젝트 폴더를 사용자 폴더로 이동

🔄 15단계: 유지보수 가이드

정기적인 업데이트

1
Node.js 업데이트 확인
node --version npm --version
2
프로젝트 의존성 업데이트
npm update npm audit fix

백업 방법

  1. 프로젝트 전체 폴더 복사하여 다른 위치에 보관
  2. 중요한 데이터는 localStorage 백업 (브라우저에서 내보내기)
  3. 설정 파일 별도 보관
📅 권장 일정:
• 매주 월요일: 시스템 상태 점검
• 매월 첫째 주: 보안 업데이트 적용
• 분기별: 전체 백업 및 복원 테스트

⚡ 16단계: 성능 최적화

시스템 최적화

  • 메모리 사용량 모니터링 - 작업 관리자에서 node.exe 프로세스 확인
  • 디스크 정리 - node_modules 정기적 재설치
  • 브라우저 캐시 관리 - 정기적인 캐시 삭제

네트워크 최적화

# 압축 활성화 npm install compression # 캐싱 설정 npm install express-static-gzip

데이터 최적화

  • localStorage 용량 관리 - 주기적으로 불필요한 데이터 정리
  • 이미지 최적화 - 도장 이미지 크기 제한
  • Excel 파일 관리 - 큰 파일 분할 처리
🎯 최적화 목표:

• 메모리 사용량 500MB 이하
